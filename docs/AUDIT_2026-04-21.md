## 6. 불필요한 fallback

### 6.1 🔴 `FillForm.tsx:332` — `portfolio.assets[assetId ?? 'sso']` 의 죽은 fallback

- **[src/components/FillForm.tsx:332](../src/components/FillForm.tsx#L332)** `{portfolio.assets[assetId ?? 'sso'] && assetId ? (...)}`
- `portfolio.assets` 는 항상 4 자산 전체 포함 (타입 `Record<AssetId, AssetSnapshot>`). `portfolio.assets[X]` 는 어떤 유효 id 에 대해서도 truthy.
- `assetId ?? 'sso'` 의 `'sso'` 폴백은 렌더링 조건에 영향 주지 않음 (뒤의 `&& assetId` 가 undefined 면 false 반환).
- 권장: `{assetId ? (<Text>... {portfolio.assets[assetId].actual_shares} ...</Text>) : null}` 로 간결화.

### 6.2 🟡 `chart.ts::mergeChartSeries` 의 `close[i] as number` / null 처리 비대칭

- **[src/services/chart.ts:30-34](../src/services/chart.ts#L30-L34)** `close: s.close[i] as number` — 타입 단언으로 null 가능성 무시. 현재 타입상 `close: number[]` 이므로 안전하지만, RTDB 가 빈 배열을 저장하지 않는 정책 때문에 인덱스가 undefined 일 가능성이 있다 (길이 불일치 시).

- 권장: 불변조건 (모든 배열 길이 동일) 을 서비스 로딩 시점에 검증 / assert.
