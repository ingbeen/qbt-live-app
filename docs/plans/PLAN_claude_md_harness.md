```
## 2. 수술적 변경 (Surgical Changes)

**필요한 것만 건드린다. 본인이 만든 흔적만 정리한다.**

기존 코드를 편집할 때:

- 인접 코드 / 주석 / 포맷을 무단으로 "개선" 하지 않는다.
- 망가지지 않은 것을 리팩토링하지 않는다.
- 본인이 다르게 썼을 스타일이라도 기존 스타일에 맞춘다.
- 무관한 dead code 가 보이면 — 삭제하지 말고 사용자에게 언급만 한다.

본인 변경으로 발생한 orphan 처리:

- 본인 변경으로 사용되지 않게 된 import / 변수 / 함수는 정리한다.
- 본인 변경 이전부터 존재하던 dead code 는 사용자 요청 없이 삭제하지 않는다.

**검증 기준**: 변경된 모든 라인이 사용자 요청과 직접 연결되는가?
```
